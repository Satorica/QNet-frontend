# 产品需求文档 (PRD)：Web 端找回密码功能

**文档状态：** 草案/待评审
**版本：** v1.0
**修订日期：** 2026-04-16

---

## 1. 项目概述
### 1.1 目标
为忘记登录密码的用户提供一个安全、便捷的自助找回流程，减少人工客服压力，提高用户留存。

### 1.2 目标用户
已注册但无法通过原密码登录 Web 站点的用户。

---

## 2. 业务流程图
**流程路径：**
1. **入口：** 登录页面 -> 点击“忘记密码”。
2. **Step 1 身份核验：** 输入邮箱 -> 获取并填写验证码。
3. **Step 2 设置新密码：** 输入符合复杂度要求的新密码 -> 再次确认。
4. **Step 3 完成：** 提示成功 -> 3秒后跳转或手动点击跳转至登录页。

---

## 3. 详细功能说明

### 3.1 第一步：身份核验 (Identity Verification)
| 字段/元素 | 类型 | 规则/校验 |
| :--- | :--- | :--- |
| **账号输入框** | Input | 必填。支持邮箱格式校验。失去焦点时校验格式。 |
| **验证码输入框** | Input | 必填。6位数字。 |
| **获取验证码按钮** | Button | 1. 点击后进入60s倒计时；2. 倒计时内禁用；3. 倒计时结束显示“重新获取”。 |

* **交互逻辑：** 如果账号未注册，前端不做强提示（防止扫描账号），仅在验证环节统一提示“账号或验证码错误”。

---

### 3.2 第二步：重置密码 (Reset Password)
| 字段/元素 | 类型 | 规则/校验 |
| :--- | :--- | :--- |
| **新密码输入框** | Password | 1. 长度 8-16 位；2. 必须包含字母和数字；3. 支持明暗码切换。 |
| **强度检测器** | Progress | 实时反馈：弱（仅数字）、中（数字+字母）、强（数字+大小写+符号）。 |
| **确认密码框** | Password | 必填。必须与“新密码”一致。失去焦点时比对。 |
| **提交按钮** | Button | 初始禁用，当以上所有校验通过后激活。 |

* **交互逻辑：** 输入不一致时，在确认密码框下方红字提示“两次输入的密码不一致”。

---

### 3.3 第三步：完成反馈 (Success View)
* **核心元素：** 成功图标、标题（密码重置成功）、副标题（请妥善保管新密码）。
* **动作：** 提供“立即登录”按钮。
* **自动化：** 页面保留 3-5 秒后，自动重定向至登录页面。

---

## 4. 安全性需求
1.  **接口限流：** 同一 IP/账号，1 分钟内只能获取 1 次验证码，24 小时内获取验证码不得超过 10 次。
2.  **有效期限制：** 验证码有效期设定为 5-10 分钟，超时失效。
3.  **防撞库设计：** 验证码连续输入错误 5 次，锁定该流程 15 分钟。
4.  **脱敏显示：** 验证环节展示用户邮箱时，需进行脱敏处理（如：138****8888）。

---

## 5. 异常流程处理 (Edge Cases)

| 异常场景 | 预期处理 |
| :--- | :--- |
| **网络中断** | 弹出 Toast 提示：“网络连接异常，请检查网络后重试”。 |
| **验证码过期** | 校验接口返回错误码后，前端提示“验证码已过期，请重新获取”。 |
| **非法访问** | 若用户未完成第一步直接访问第二步 URL，系统应重定向回第一步。 |
| **重复使用** | 找回密码的临时 Token 在重置成功一次后立即失效。 |
